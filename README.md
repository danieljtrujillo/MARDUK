MARDUK
Mamba-Augmented Reconstruction & Decoding of Unknown Kuneiform
A machine translation system that converts ancient Akkadian cuneiform transliterations into English, built for the Kaggle "Deep Past Initiative" competition ($50K prize pool).

What This Project Does
This codebase tackles one of the most challenging problems in computational linguistics: automatically translating 4,000-year-old Akkadian texts (the oldest known Semitic language, written in cuneiform script) into modern English. The system takes as input scholarly transliterations of cuneiform tablets — stylized Latin-character representations of Akkadian signs with specialized notation for damaged, missing, or uncertain readings — and produces fluent English translations. It combines two complementary approaches: (1) a hybrid neural seq2seq model pairing a bidirectional Mamba state-space encoder with a Transformer decoder, trained on ~8,500 parallel examples with auxiliary objectives for named entity, number, and damage-marker recognition; and (2) a retrieval-augmented LLM pipeline that uses TF-IDF similarity search, a 39K-entry Akkadian lexicon, a 19K-entry dictionary, genre detection, and sibling-tablet context to prompt Claude for high-quality translations. The neural model runs offline within Kaggle's 9-hour GPU constraint as a code competition submission, while the LLM pipeline serves as a development and benchmarking tool. Both approaches feed into a shared evaluation framework scored by the geometric mean of BLEU and chrF++, the competition's official metric.

Model Architecture (Hybrid Mamba + Transformer)
Source (Akkadian transliteration, byte-encoded)
    │
    ▼
  Dual-View Packing: <raw> ... </raw> <norm> ... </norm>
    │
    ▼
  Bidirectional Mamba SSM Encoder (8 layers, 512d)
  ├── Forward Mamba + Backward Mamba → Concatenate → Project
  └── Fallback: Bi-GRU if mamba_ssm not installed
    │
    ▼
  Transformer Decoder (4 layers, 8 heads, 512d)
  ├── Cross-attention to encoder memory
  └── T5-small target tokenizer (32,100 vocab)
    │
    ├─→ Translation head (main loss)
    ├─→ Name span head (BIO tags, lexicon-aware)
    ├─→ Number span head (BIO tags)
    └─→ Damage span head (BIO tags)
    │
    ▼
  Beam Search (5 beams, length penalty 0.9, n-gram blocking)
    │
    ▼
  submission.csv
Model: 59.3M parameters | Training: Cosine annealing LR with warmup, GroupKFold (no tablet leakage)

RAG / LLM Pipeline Architecture
Input (transliteration) 
    │
    ├─→ TF-IDF char n-gram index → Top-7 similar training pairs
    ├─→ 39K-entry OA Lexicon → Per-word type/normalized/lexeme
    ├─→ 19K eBL Dictionary → English definitions
    ├─→ Phrase Memory → Common OA formulae translations
    ├─→ Genre Detector → seal/debt/letter/financial/witness
    ├─→ Sibling Context → Other segments of same tablet
    │
    ▼
  Claude API (Sonnet, temp=0.15)
  + Domain-specific system prompt
  + 7 RAG few-shot examples
  + Dictionary augmentation
    │
    ▼
  Post-processing → submission.csv

Quick Start
Neural Model (Kaggle Submission)
# 1. Set up environment
python -m venv .venv && source .venv/bin/activate  # or .venv\Scripts\activate on Windows
pip install -e .

# 2. Prepare data (normalize, pack dual views, create folds)
python -m src.data.prepare \
    --data-config configs/data/raw.yaml \
    --view-config configs/data/dual_view.yaml

# 3. Train hybrid model
python -m src.train.train_hybrid \
    --data-config configs/data/raw.yaml \
    --model-config configs/model/mamba_enc_txd_dec_base.yaml \
    --train-config configs/train/hybrid.yaml

# 4. Generate submission
python -m src.eval.decode \
    --data-config configs/data/raw.yaml \
    --view-config configs/data/dual_view.yaml \
    --model-config configs/model/mamba_enc_txd_dec_base.yaml \
    --checkpoint outputs/runs/hybrid_base/best.pt \
    --output submission.csv

Key Design Decisions
Byte-level source encoding: Handles Akkadian's special Unicode characters (ḫ, ṣ, ṭ, š) and subscript/superscript notation without subword tokenization artifacts
Dual-view packing: Model sees both raw transliteration and normalized form side-by-side, letting it learn from orthographic variation while grounding on canonical forms
Bidirectional Mamba encoder: State-space model captures long-range dependencies in cuneiform texts more efficiently than vanilla Transformers, with forward+backward passes for full context
Lexicon-aware BIO labels: Auxiliary heads trained on proper noun, number, and damage spans identified via the 39K OA Lexicon — helps the model learn domain structure
GroupKFold splitting: Prevents data leakage from sibling sentences of the same tablet appearing in both train and validation sets
Character n-gram TF-IDF (RAG pipeline): Handles Old Assyrian orthographic variation where the same word appears with different spellings across tablets
Geometric mean awareness: Both pipelines optimize for balanced BLEU and chrF++ since the metric punishes weakness in either
